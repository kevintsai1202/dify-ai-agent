# Day 1: Dify 工作流與 Claude 雲端備課實戰工作坊 - 課程大綱

## 學習目標
- 能夠在無需本地端環境建置的情況下，使用網頁版 Claude Team 帳號進行結構化提示詞備課。
- 理解 Model Context Protocol (MCP) 的運作機制，學會在網頁端配置 Canva MCP 進行簡報生成。
- 能夠運用 Claude Artifacts 快速生成可互動的 HTML 課程網頁簡報並下載。
- 理解 Dify 雙層知識庫（院級共用庫與老師個人庫）的架構，並完成個人科系教材上傳向量化。
- 能夠操作 Dify 設計思考工作流 WebApp，自動檢索院級知識與個人教材，並由 LLM 合併產出「同理、定義、發想」三階段教學活動設計。
- 結合個人專業教學判斷，完成下學期課程之教案草稿，並反思教師如何引導學生善用 AI 發展創意、判斷力與未知領域探索能力。

## 單元細部

### d1-u1: 為什麼 AI 能幫你備課
- **學習目標**: 能夠登入 Claude Team，掌握基礎結構化提示詞，自動產出教案草稿與評分標準，建立 AI 備課信心。
- **任務 (tasks)**:
  - 登入 Claude Team 官方網頁版帳號
  - 輸入自己下週或下學期的真實課程主題
  - 使用日常備課 Prompt 引導 Claude 生成教案大綱、課堂討論題與學生作業評分說明
  - 微調提示詞進行二輪優化，感受 AI 快速產出備課素材的成效
- **素材需求**: Claude Team 官方帳號、日常備課與教案草稿生成 Prompt、講義《Claude_Team_Quickstart.md》。

### d1-u2: 什麼是 MCP？（概念說明）
- **學習目標**: 理解 Model Context Protocol (MCP) 的運作機制，了解 AI 如何透過標準介面調用外部服務以擴充能力。
- **任務 (tasks)**:
  - 聆聽講師說明 MCP（Model Context Protocol）的運作概念（比喻為幫 AI 裝上操作外部工具的手）
  - 理解 MCP 與傳統 API 串接的差異：MCP 提供標準介面，讓 AI 自主理解並操作外部應用
  - 了解工作坊中我們如何透過網頁端開啟 Canva MCP 連接，實現在 Claude 中直接建立與編輯簡報
- **素材需求**: MCP 運作架構圖與比喻投影片。

### d1-u3: 用 AI 生成課程簡報（雙路徑實作）
- **學習目標**: 能夠完成 Canva MCP 連接授權，學會使用 Canva MCP 在 Canva 建立投影片，或利用 Claude Artifacts 快速生成互動網頁簡報。
- **任務 (tasks)**:
  - 在 Claude 設定頁面中開啟 Canva 連接權限，完成授權
  - **路徑一（Canva MCP）**：在對話中調用 Canva MCP，指定模板風格並直接在 Canva 建立投影片，並匯出為 PPTX
  - **路徑二（Claude Artifacts）**：在 Claude 中輸入課程簡報 Prompt，利用 Artifacts 生成可互動的 HTML 網頁簡報，並下載至桌面以備離線投影使用
  - 每位老師產出一份自己科系的課程簡報，並下載至電腦中
- **素材需求**: Canva 簡報 Prompt、Artifacts 簡報 Prompt、講義《Canva_MCP_Integration.md》與《Claude_Artifacts_Guide.md》。

### d1-u4: 建立個人知識庫
- **學習目標**: 理解 Dify 雙層知識庫的架構，學會登入 Dify 團隊版平台並上傳個人科系教材進行向量分段與儲存。
- **任務 (tasks)**:
  - 理解兩層知識庫設計：院級共用庫（提供設計趨勢、系所特色、全院共享規範）與老師個人庫（存放老師自己的教案與教材）
  - 登入 Dify.ai 團隊版平台，將自己的 1-2 份課程教材 (PDF / Word) 上傳至個人知識庫，完成向量分段與儲存
  - 理解工作流預先串接的雙路檢索節點：AI 產出時會自動在背景分別參照院級共用庫與老師個人庫，再交由 LLM 合併輸出，老師不需手動串接
- **素材需求**: Dify 團隊版帳號、1-2 份教師個人的教學大綱或講義、講義《Dify_Knowledge_Base_Setup.md》。

### d1-u5: 體驗設計思考輔助 AI Agent（Dify 工作流實作）
- **學習目標**: 理解設計思考中「同理、定義、發想」三階段在教學活動設計上的邏輯，並學會在 Dify 工作流網頁端中，實作雙知識庫檢索、LLM 合併與生成三階段內容。
- **任務 (tasks)**:
  - 【講師演示】聆聽講師說明 Dify 工作流設計邏輯：輸入節點、院級庫檢索、個人庫檢索、LLM 合併節點如何影響產出（理解即可，不需要操作）
  - 【同理階段實作】：在工作流 WebApp 中輸入課程主題，系統自動檢索個人教材與院級資料，產出學習者痛點、訪談問題建議與同理心地圖框架
  - 【定義階段實作】：系統自動彙整前述同理資料，定義 HMW (How Might We) 問題框架與教學目標
  - 【發想階段實作】：系統參照院級共用庫與老師個人庫，快速生成 20+ 個創意教學活動點子
  - 確認產出確實受到兩層知識庫影響，體驗兼具學院脈絡與個人課程特色的 AI 備課大腦
- **素材需求**: 講師預建並發布的 Dify 工作流 WebApp 網頁連結、講義《Dify_Workflow_Design_Thinking.md》。

### d1-u6: 教案草稿實作與反思交流
- **學習目標**: 能夠選定課程主題，使用 Dify 工作流網頁端跑完三階段，產出完整下學期教案草稿與教學活動建議，並反思 AI 成品快速提升後的課堂引導方式。
- **任務 (tasks)**:
  - 選定一個下學期的課程主題，在 Dify 工作流網頁端跑完三階段設計思考流程，產出初步教學活動建議
  - 對照原本的備課方式，記下 AI 已經能明顯提升成果品質的地方，以及仍需要教師判斷、取捨與價值引導的地方
  - 分享交流：每位老師進行 3 分鐘的心得分享，提出如何引導學生用 AI 拓展創意、探索未知問題，而不是只交出快速生成的成品
  - 填寫反思回饋表單，完成本日工作坊學習記錄
- **素材需求**: 反思回饋表單連結、講義與表單《Workshop_Reflection_Template.md》。
