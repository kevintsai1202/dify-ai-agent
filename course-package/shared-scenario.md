# 2026 課程共用案例：設計思考與 AI 備課的教學應用實務

## 1. 案例背景與教學升級情境

*   **角色設定**：趙總監 (Director Zhao) — 一位資深的服裝設計大師兼任設計時尚學院的講師。他對複雜的程式代碼、Git 命令行與環境變數設定完全不熟悉，但面臨著多重教學與課務的真實痛點。
*   **面臨痛點**：
    1.  需要在短時間內，根據 **2026 年最新流行色彩趨勢**（例如：變革深藍綠、雲舞暖白等），為下學期的課程編寫完整的教案大綱、課堂討論題與作業評分標準。
    2.  需要製作精美的上課投影片，但製作簡報耗時費力，且傳統 AI 生成的簡報往往缺乏美感約束。
    3.  希望將科系內共用的「設計趨勢參考資料」與自己多年累積的「個人教案教材」結合，建立一個個人的 AI 備課大腦。
    4.  希望利用設計思考方法（同理、定義、發想）規劃出真正能引發學生興趣的創意課堂互動活動，但苦於缺乏時間與點子。

*   **一站式無代碼解決方案**：
    趙總監以 **Claude 線上版 (Claude.ai)** 與 **Dify 工作流 WebApp** 作為核心操作介面，一站式解決了以上痛點：
    1.  **日常備課**：在 Claude 網頁版中直接用中文輸入「色彩搭配學」課程主題，快速產生教案草稿、討論題與作業評分標準。
    2.  **連接 Canva MCP**：在 Claude 中開啟 Canva 連接授權，下達 Prompt 調用 Canva MCP 直接在 Canva 後台建立一份精美的色彩趨勢簡報，並匯出 PPTX 檔案；或者利用 **Claude Artifacts** 產生可互動 HTML 投影簡報。
    3.  **建立 Dify 雙知識庫**：學院端預先建立院級共用庫，存放設計趨勢、系所特色與共同評量規範；趙總監在 Dify 平台將個人教材 PDF 檔案上傳到老師個人知識庫。
    4.  **在 Dify WebApp 跑設計思考實作**：趙總監在講師預建的 Dify 工作流 WebApp 中輸入課程主題。工作流會分別檢索院級共用庫與老師個人庫，再由 LLM 合併與去衝突，跑完同理（分析學生痛點）、定義（定義 HMW 與教學目標）、發想（發想 20+ 個創意教學點子）三階段，一站式完成教案生成。

## 2. 教師個人課程主題之任務對應

在工作坊中，老師們將扮演類似「趙總監」的角色，直接以**「自己下學期要上的真實課程」**（如：視覺傳達設計、數位媒體製作、流行設計實務等）為核心，在 Claude.ai 介面中對應執行以下任務：

*   **任務一 (第一小時)**：在 Claude 網頁版輸入個人課程主題，產出教案草稿與評分標準。
*   **任務二 (第二小時)**：在 Claude 中連接 Canva MCP，生成個人課程的簡報，或用 Artifacts 生成網頁投影投影片。
*   **任務三 (第三小時)**：在 Dify 上傳科系教材建立老師個人知識庫，並理解院級共用庫與個人庫的資料放置原則。
*   **任務四 (第四小時)**：在 Dify 工作流 WebApp 跑設計思考三階段，確認 LLM 如何合併院級知識與個人教材，完成個人真實教案的修改與產出，並進行反思交流。

## 3. 專案數據與設計 Token 範例

在第三與第四小時的 Dify 知識庫實作中，我們將使用以下的 `DESIGN.md` 設計規格與工作流資料作為示範，方便老師們理解個人知識庫的檔案結構。

### 設計 Token (YAML 格式) — 品牌視覺與排版規範範例
```yaml
---
theme: studio-2026
colors:
  primary: "#004f54"      # Transformative Teal (變革深藍綠)
  secondary: "#f4f1ea"    # Cloud Dancer (雲舞暖白)
  accent: "#ee9b00"       # Acacia Gold (金合歡金黃)
  neutral-dark: "#141e1e" # 深炭綠 (文字)
typography:
  serif: "Georgia, serif"
  sans: "Microsoft JhengHei, sans-serif"
spacing:
  margin-default: "24px"
  section-padding: "120px"
---
```

### Dify RAG 工作流的資料檢索結構 (JSON)
```json
{
  "inputs": {
    "topic": "個人課程主題 (例如: 包浩斯設計史)",
    "student_level": "大二設計系學生",
    "stage": "同理/定義/發想"
  },
  "retrieval": {
    "college_knowledge_base": "設計趨勢、系所特色、共同評量規範",
    "teacher_knowledge_base": "個人課綱、講義、歷年作業與評分習慣"
  },
  "merge_rule": "院級規範優先作為邊界；個人教材補足案例、活動細節與課堂語氣。",
  "response": {
    "text": "這是透過 Dify 工作流檢索並合併生成的教學活動點子與 HMW 框架...",
    "referenced_documents": ["college_kb/design_trends.pdf", "teacher_kb/my_syllabus.pdf"]
  }
}
```
這個引導式案例將貫穿整個半天工作坊，讓所有無資訊背景的老師都能在直覺的操作中，深刻體會到如何以 Claude 進行日常備課與簡報生成，並以 Dify WebApp 執行雙知識庫檢索與 LLM 合併輸出，極大地簡化備課流程，並產出高品質的教學成果。
