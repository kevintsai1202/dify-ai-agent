# 課程規劃案：「設計學院 AI 工具應用工作坊」

本課程專為**設計時尚學院**教師（視覺傳達、數位媒體、創意產品、流行設計、服飾設計、時尚經營等科系）量體打造。考慮到教師多數無資訊背景，課程設計採用最直覺的**「網頁端操作與雲端工作流整合」**架構。

為了確保現場實作的順暢度，並避免 4 小時工作坊因環境設定卡關而上不完，本課程**「全面免去本地端繁瑣環境建置（如 Git, Node.js, Python 等下載與環境變數設定）」**，也**「捨棄在終端機輸入命令列進行 Skills 安裝的步驟」**，改以直覺的**瀏覽器網頁端**工具進行操作。

工作坊以 **Claude 線上版 (Claude.ai) 與 Dify.ai 雲端平台為核心操作介面**，透過配置兩個主要的網頁端工具——**Canva** (Canva MCP 簡報生成，第二小時) 與 **Dify** (Dify 雲端知識庫與設計思考工作流，第三、四小時)——讓老師們可以在 Claude 的聊天室中調用 Canva 簡報功能，並在 Dify 的工作流網頁端中檢索與分析科系教材，跑完設計思考備課流程。

---

## 📌 課程基本資訊

*   **對象**：設計時尚學院教師（共 10 位，無資訊背景）
*   **時長**：4 小時（半天工作坊）
*   **核心工具**：Claude Team (網頁版 + Artifacts)、Canva (搭配 MCP 整合)、Dify (團隊版雲端後台，搭配設計思考工作流 WebApp)
*   **教學目標**：
    1.  理解 AI 在日常備課、教案生成及評分標準設計上的應用，掌握結構化提示詞。
    2.  理解 MCP (Model Context Protocol) 概念，學會將 Claude 帳號與外部工具（Canva）連接。
    3.  利用 Claude Artifacts 快速生成可互動的 HTML 網頁簡報，或透過 Canva MCP 自動產生美感投影片並匯出為 PPTX。
    4.  學會 Dify 雙層知識庫的配置，完成個人教材上傳，理解院級共用知識與老師個人知識如何分開放置、分別檢索並由 LLM 合併輸出。
    5.  在 Dify 設計思考工作流網頁端中，體驗以「同理、定義、發想」三階段設計思考輔助 AI 產出量身打造的創意教學活動，感受串接院級知識與個人知識後的產出差異。
    6.  獨立產出下學期課程的教學活動設計教案，並完成教學應用的反思與回饋。

## ⏱️ 4 小時課程大綱

### 🕒 第一小時：AI 備課入門與 MCP 工具連接
*   **主題**：*「無痛起步：體驗 Claude 備課超能力與 MCP 工具概念」*
*   **授課模式**：講師演示 + 教師跟隨操作
*   **時間分配**：
    *   **13:00 - 13:45 (45分鐘)：為什麼 AI 能幫你備課（Claude Team 實作體驗）**
        *   **破冰**：邀請每位老師分享目前備課最花時間的一件事。
        *   **講師示範**：在 Claude Team 網頁版輸入下週課程主題，一鍵生成教案草稿、課堂討論題與學生作業評分說明。
        *   **自由實作**：老師使用自己下週或下學期真實的課程內容親手跑一次。
        *   *【重點】* 此階段不涉及任何 Agent 複雜操作，旨在消除技術恐懼、建立操作信心。
    *   **13:45 - 14:00 (15分鐘)：什麼是 MCP？（概念說明與連結）**
        *   **MCP (Model Context Protocol) 說明**：比喻 Claude 是一個很聰明的大腦，而 MCP 就是幫他裝上不同的手，讓他可以操作外部工具（如 Canva）。
        *   **工具展望**：展望我們在後續小時如何透過網頁端，操作 Canva（簡報生成）與 Dify（設計思考工作流）。

---

### 🕒 第二小時：AI 簡報生成與設計美感落地
*   **主題**：*「高顏值簡報：利用 Artifacts 與 Canva MCP 快速生成教材」*
*   **授課模式**：講師操作示範 + 教師跟隨體驗
*   **時間分配**：
    *   **14:00 - 14:45 (45分鐘)：用 AI 生成課程簡報（雙路徑實作）**
        *   **路徑一：Canva MCP（圖文精美、可編輯）**
            *   講師示範：在 Claude 設定頁面中開啟 Canva 連接授權。在對話中調用已連接的 Canva MCP，命令 Claude 直接在 Canva 後台建立一份 8 頁的投影片，套用美感模板、加上流行色彩配色意境，最後直接匯出為 PPTX 簡報檔。
        *   **路徑二：Claude Artifacts（快速、互動、可離線）**
            *   講師示範：輸入大綱，讓 Claude 自動在側邊欄產生一個極簡、有設計感、可互動的 HTML 網頁簡報，並下載固定色彩的 HTML 檔案以備離線投影使用。
        *   **跟隨實作**：每位老師選擇一種路徑，產出自己專屬科系的課程簡報，並下載至電腦中。
    *   **14:45 - 15:00 (15分鐘)：休息與自由提問**

---

### 🕒 第三小時：Dify 雲端知識庫與 Dify 工作流實作
*   **主題**：*「雲端知識庫配置與設計思考工作流體驗」*
*   **授課模式**：講師演示 + 教師跟隨操作
*   **時間分配**：
    *   **15:00 - 15:20 (20分鐘)：建立個人備課知識庫（Dify 知識庫設定）**
        *   **知識庫架構**：說明兩層知識庫設計——「院級共用庫」（提供設計趨勢、系所特色、共同評量規範，全院共享）與「老師個人庫」（存放老師自己的教案、講義與評分習慣）。
        *   **實作步驟**：引導老師們將自己的 1-2 份課程教材 (PDF / Word) 上傳至 Dify 個人知識庫，完成向量分段儲存。
        *   **合併輸出邏輯**：講師說明預建工作流會以同一個課程主題分別檢索院級庫與個人庫，再由 LLM 依「院級規範優先、個人教材補細節」的原則合併輸出。
    *   **15:20 - 16:00 (40分鐘)：體驗設計思考輔助 AI Agent（Dify 工作流實作）**
        *   **開場引導**：講師打開 Dify 後台說明工作流設計邏輯（輸入節點、院級庫檢索、個人庫檢索、LLM 合併節點如何對應同理、定義、發想，以及知識庫檢索如何影響產出），老師理解即可，不需要跟著操作。
        *   **工作流實作**：引導老師在預建並發布的工作流 WebApp 中輸入自己科系的課程主題，親手跑過「同理、定義、發想」三階段：
            *   *同理階段*：輸入主題，系統自動檢索個人教材與院級共用資料，產出學習者痛點與同理心地圖。
            *   *定義階段*：系統彙整資料，定義 HMW (How Might We) 問題與教學目標。
            *   *發想階段*：系統參照個人庫與共用庫，快速生成 20+ 個創意教學活動點子。
        *   **檢索測試**：確認產出確實受到個人知識庫教材的影響，體會專屬 AI 備課大腦的成效。

---

### 🕒 第四小時：教案草稿實作與教學應用反思
*   **主題**：*「設計思考教案深耕與實務應用交流」*
*   **授課模式**：教師網頁端實作 + 講師輔導 + 互動交流分享
*   **時間分配**：
    *   **16:00 - 16:40 (40分鐘)：教案草稿實作（各科系套用設計思考三階段）**
        *   **實作任務**：每位老師選定一個下學期的課程主題，使用 Dify 工作流網頁端跑完三階段，產出教案草稿與教學活動建議。
        *   **專業判斷介入**：強調 AI 產出的是教案的「原料」，老師在對話中微調與篩選。記錄「AI 幫到你的地方」與「你覺得 AI 還不夠的地方」。
        *   **一對一輔導**：講師與協辦人員現場巡迴，協助解答老師在 Dify 平台操作上的個別問題。
    *   **16:40 - 17:00 (20分鐘)：分享與填寫反思回饋（交流座談）**
        *   **心得分享**：每人約 3 分鐘，分享在網頁端透過 Canva 與 Dify 工作流備課的實務體驗與反思。
        *   **線上反思**：引導老師們填寫線上反思回饋表單。

## 🛠️ 課前準備與教材提示（學校承辦人與教師準備清單）

為了讓工作坊的實作進行得更順暢，建議課前完成以下準備：

1.  **平台帳號預先註冊**：
    *   請老師於課前先註冊好 **Dify.ai 團隊版帳號** 與 **Claude.ai 帳號**，以省下現場註冊與收信驗證的時間。
2.  **教學素材準備**：
    *   請每位老師攜帶 **3-5 份自己的教學大綱 PDF** 或 **歷年課程講義**。
    *   這些資料將在第三小時被上傳至 Dify 個人知識庫，作為 AI 生成教案的專屬語境。
